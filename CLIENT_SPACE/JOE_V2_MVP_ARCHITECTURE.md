- Telegram-бот (Python, Railway)
- Claude API для скоринга JD
- Google Sheets как БД (77 строк × 21 колонка)

## 2. Цель Joe v2

Автономный агент для Андрея с:
- Парсингом пачки LinkedIn-ссылок
- Наглядным UI (таблица + карточки + дашборд)

**MVP-принцип:** Проект эволюционирует, никогда не конечный. MVP = фундамент для роста.

## 4. Режимы MVP

### Включены

| Режим | Что делает |
|-------|-----------|
| **CV_TAILOR** | Адаптация резюме: акценты в summary + навыки + 2 последних места. Показывает изменения → аппрув |
| **LETTER** | Письмо 150-220 слов. Тема + текст. Имя из JD если есть контакт |

| FORM | Позже |

| BASE_CV выбор (US/INTL) | Одно каноническое резюме |

## 5. Архитектура: CLIENT_SPACE

Универсальная структура профиля клиента (Андрей первый, но заменяемый):

```
CLIENT_SPACE/
├── canonical/
│   ├── resume.md            # единственный источник правды
│   ├── facts.md             # цифры, даты, достижения
│   └── allowed_phrases.md   # допустимые формулировки
│
├── strategy/
│   ├── target_markets.md    # гео, индустрии
│   ├── target_roles.md      # позиции, уровни
│   └── constraints.md       # что НЕ рассматриваем
│
├── process/
│   ├── letter_template.md   # структура письма + примеры
│   └── rules.md             # правила откликов
│
└── snapshots/
    └── [timestamped versions]
```

### Правила Space
- Space активен всегда (не прикрепляется вручную)
- Joe сверяет любые утверждения с Space
- Нет данных в Space = неизвестно
- Обновление только через Snapshot
- ❌ домысливать факты
- ❌ улучшать цифры
- ❌ отходить от канона

## 6a. Модульная архитектура

### Принцип: модули как plug-in

```
JOE_CORE (ядро)
    │
    ├── CV_MODULE
    │   └── cv_rules.md
    │
    ├── LETTER_MODULE ← легко заменяемый
    │   ├── letter_rules.md      (стилистика)
    │   ├── letter_examples.md   (примеры)
    │   └── banned_phrases.md    (запреты)
    │
    └── SCORING_MODULE
        └── scoring_rules.md     (fit/flags)
```

### Letter Module — требования

| Требование | Реализация |
|------------|------------|
| Легко редактировать стиль | Отдельный `letter_rules.md` |
| Легко добавить примеры | Отдельный `letter_examples.md` |
| Полностью заменить | Замена папки `/modules/letter/` |
| Не ломает остальное | Изолированный интерфейс: `generate_letter(jd, canon, notes) → letter` |

### Файловая структура модулей

```
/modules
  /cv
    config.md
    tailor.py
  /letter
    config.md        ← JOE_Process_Letter_v2.md
    generate.py
  /scoring
    config.md
    evaluate.py
```

## 9. UI/UX Specification

### Карта экранов

```
[PIPELINE] → [КАРТОЧКА] → [CV] → [LETTER] → [ГОТОВО]
     ↑                                          │
     └──────────────────────────────────────────┘
                    
[DASHBOARD] — отдельно в меню
```

### Экран 1: PIPELINE

| Элемент | Описание |
|---------|----------|
| Вид | Таблица |
| Колонки | Компания, Роль, Статус, Дата, Контакт, Письмо (truncated) |
| Сортировка | По статусам: Новая → Подано → Отправлено |
| Письмо | Первые ~50 символов, клик → popup с полным текстом |
| Клик на строку | → Карточка вакансии |
| Будущее | Фильтры (учесть в архитектуре) |

### Экран 2: КАРТОЧКА ВАКАНСИИ

| Элемент | Описание |
|---------|----------|
| Показывает | JD (текст), контакт если есть, скор Joe (go/no-go), объяснение |
| Действия | `[Подготовить отклик]` → CV / `[Skip]` → Pipeline |

### Экран 3: CV (track changes)

| Элемент | Описание |
|---------|----------|
| Вид | Каноничное резюме с правками |
| Правки | Выделены цветом (как track changes) |
| Действия | `[Скачать PDF]` / `[Копировать правки]` / `[Далее → Letter]` |
| Копировать правки | Только изменения текстом — для вставки в свой файл |

### Экран 4: LETTER

| Элемент | Описание |
|---------|----------|
| Поле ввода | Опционально: "Что подчеркнуть?" (пожелания к письму) |
| Логика | Пусто → генерится по канону / Есть ввод → канон + пожелания |
| Результат | Subject + текст письма |
| Действия | `[Скачать PDF]` / `[Копировать]` / `[Готово]` |

### Экран 5: DASHBOARD

| Элемент | Описание |
|---------|----------|
| Статистика | Всего / Новые / Подано / Отправлено / Ответы |
| Визуал | Карточки с цифрами, воронка (простая) |
| MVP | Минимум, расширяем позже |

### UX-принцип

Один экран = одно решение. Не думать "куда положить всё", думать "что человек делает прямо сейчас".

## 10. Архитектура данных

### Источник правды: Google Sheets

```
Google Sheets (единственная БД)
         ↓
    [Кэш в памяти]
         ↓
    Web-интерфейс
         ↑
    Telegram-бот (пишет)
```

### Принципы
- Читаем из Sheets при загрузке
- Пишем в Sheets при каждом действии
- UI показывает из кэша (быстро)
- Аналитика строится по Sheets напрямую

### Структура колонок (дополнение к текущим)

| Колонка | Содержимое |
|---------|------------|
| `cover_letter` | Полный текст письма (многострочный) |
| `cover_letter_notes` | Пожелания к письму (опционально, пока не храним) |
| `cv_changes` | Текст изменений в резюме |

### Отображение письма в Pipeline
- В таблице: первые ~50 символов (truncate)
- По клику: popup с полным текстом
- Таблица остаётся компактной

## 11. Business Rules & Triggers

### Тайминги

| Событие | Когда | Действие |
|---------|-------|----------|
| **Stale New** | New + 3 дня | Напоминание разобрать |
| **Follow-up 1** | Applied + 4 дня | Напоминание |
| **Follow-up 2** | Follow-up 1 + 7 дней | Второе напоминание |
| **No Response** | Applied + 30 дней | Авто-статус или напоминание закрыть |

### Флаги и маркеры

| Тип | Флаг | Что значит | Блокирует? |
|-----|------|------------|------------|
| 🔴 **Stop** | `visa_required` | США + явно требуется рабочая виза | Да |
| 🔴 **Stop** | `citizenship` | Требуется гражданство (EU/другое) | Да |
| 🟡 **Warning** | `exp_gap` | Часть требуемого опыта отсутствует (показать %) | Нет — подсветить |
| 🟡 **Warning** | `junior_role` | Ищут ≤5 лет опыта | Нет — подсветить |
| 🔴 **Review** | `strong_mismatch` | Сильное несовпадение (тип роли, домен) | Не блок, но красный маркер → человек решает |

**Логика флагов:**
- 🔴 Stop → вакансия не добавляется (или добавляется с блоком)
- 🟡 Warning → добавляется, подсвечивается, человек видит
- 🔴 Review → добавляется с красным маркером, требует ручного решения (взять / удалить)

**Что НЕ является флагом:**
- Локация / релокация — не блокирует, не предупреждает

**Не флаг:** локация/релокация — просто информация, не предупреждение.

### Проверка дубликатов

| Проверка | Как |
|----------|-----|
| По ссылке | Exact match URL |
| По Company + Role + Region | Fuzzy match (если ссылки разные, но вакансия та же) |
| Действие | Показать "возможный дубль" → человек решает |

### Статусы и переходы

```
New → In Progress → Applied → Waiting → Response / No Response
                                           ↓
                                      Interview → Closed
```

| Статус | Значение | Кто ставит |
|--------|----------|------------|
| `New` | Вакансия добавлена, не обработана | Система |
| `In Progress` | Готовим отклик | Человек (клик "Подготовить") |
| `Applied` | Отклик отправлен | Человек (после "Готово") |
| `Waiting` | Ждём ответ | Система (Applied + 4 дня авто) |
| `Response` | Есть ответ (любой) | Человек |
| `Interview` | Назначено интервью | Человек |
| `No Response` | Тишина > 30 дней | Система или человек |
| `Closed` | Финал (offer/reject/withdrawn) | Человек |

### Reconnect (будущее)

| Параметр | Значение |
|----------|----------|
| Когда предлагать | 90 дней после No Response/Rejected + обновился CV |
| MVP | Ручное решение |
| Позже | Флаг в UI "можно переоткликнуться" |
