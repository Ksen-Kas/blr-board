# MIA · Content System · V1
Status: MVP · in progress

## 1. Цель этого мира

Создать работающую контент-систему для Мии, в которой:
- голос Мии собран и понятен системе,
- Soft Frame Agency пишет тексты в этом голосе,
- контент-план можно выполнять без постоянного ручного “придумывания с нуля”,
- в будущем система станет агентом полного цикла (от плана до CRM), но не сразу.

## 2. Модули в ЭТОЙ версии (MVP)

**Включены сейчас:**
- `VoiceEngine (Ayra)` — сбор и картридж голоса Мии.
- `SoftFrameAgency (SFA)` — контент-агентство на базе SFA_Module_v2.0.
- `UCM / Living Profile` — живой профиль для смысла и философии.

**Отложены на потом:**
- CRM-оператор (ответы клиентам).
- Авто-контент-планер на основе метрик.
- Любые внешние интеграции.

## 3. Файлы в проекте

- `MIA_LivingProfile_RAW.md` — сырой живой профиль Мии (мы можем собрать его заново).
- `MIA_VoiceCorpus_RAW.md` — живой голос Мии (только её тексты).
- `core_modules_archive/SFA_Module_v2.0.yaml` — паспорт агентства SoftFrameAgency.
- (опционально) `MIA_ChatArchive_pre-2025-11-20.md` — архив части старых диалогов, если решим его делать.

## 4. Как мы работаем в этом мире (MVP)

1. **Собираем голос**  
   — наполняем `MIA_VoiceCorpus_RAW.md` 5–7 фрагментами живых текстов Мии.

2. **Обновляем профиль**  
   — из `MIA_LivingProfile_RAW.md` UCM собирает актуальный живой профиль (v1).

3. **Запускаем SFA на этом голосе и профиле**  
   — SFA пишет первые тексты по простым задачам или базовому контент-плану.
   — ты даёшь корректировки, мы фиксируем поведение.

4. **Получаем первую рабочую неделю текстов**  
   — не идеальную, но стабильную и живую.

## 5. Что важно помнить

- Этот мир Мии — **про систему**, а не только про “красивые тексты”.
- Мы не тянем сюда весь старый хаос: только то, что помогает собрать голос и смысл.
- Старые ветки (Ayra 2.6, упаковка, живой профиль) — это архив и сырьё, а не истина.
- Всё, что здесь лежит, — может обновляться, но SFA_Module_v2.0.yaml остаётся опорным модулем.
