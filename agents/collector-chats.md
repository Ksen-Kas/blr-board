## Что делать

1. Для каждого чата: `tg_get_messages(group=..., limit=30)` (последние 30 сообщений)
2. Из сообщений извлеки:
   - **Задачи** — что нужно сделать (явное или подразумеваемое)
   - **Решения** — что было решено в обсуждении
   - **Дедлайны** — упомянутые даты
   - **Churn-сигналы** — кто замолчал, кто жалуется, кто уходит
   - **Wins** — кейсы, благодарности, артефакты участников
3. Для каждой извлечённой задачи создай объект inbox:
   ```json
   { "id": "i{порядковый номер}", "title": "краткое название", "sub": "контекст: откуда, от кого, когда" }
   ```
4. Прочитай текущий `data.json` из `/Users/sizovaka/Documents/AI_LAB/GitHub/blr-board/data.json`
5. Добавь новые элементы в массив `inbox` (НЕ заменяй существующие)
6. Обнови поле `updated` на текущую дату
7. `git add data.json && git commit -m "collector: inbox update" && git push`

## Правила

- НЕ дублируй задачи которые уже есть в fire/tasks/backlog
- НЕ добавляй шум (приветствия, мемы, бытовые разговоры)
- Формулируй title как ЗАДАЧУ (что нужно сделать), не как факт
- sub должен содержать: кто сказал, в каком чате, примерная дата
- Если сообщение — это win/кейс участника, пометь в sub: "[win]"
- Если сообщение — churn-сигнал, пометь в sub: "[churn]"
- Максимум 10 новых элементов за один проход (не перегружай inbox)

## MCP Tools

Используй `tgmcp-read`:
- `tg_get_messages(group="...", limit=30)` — получить сообщения
- `tg_get_message_count(group="...")` — проверить активность
